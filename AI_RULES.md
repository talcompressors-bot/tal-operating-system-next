# AI RULES

## Source Of Truth

1. Always check Project Brain before asking for screenshots.
2. Always check GitHub documentation before suggesting architecture changes.
3. Never assume project structure.
4. If information exists in Project Brain, use it first.

## Development Rules

1. Never modify Production directly.
2. All changes must be tested first.
3. Prefer small isolated changes.
4. Every change must be reversible.
5. Avoid creating new functions when existing functions can be reused.

## AppSheet Rules

1. Check for Bot conflicts.
2. Check for duplicate executions.
3. Check for status loops.
4. Check for AppSheet and Apps Script updating the same row.
5. Check for duplicate document creation.

## AI Review Rules

Before approving any change:

- Explain what changed.
- Explain risks.
- Explain impacted functions.
- Suggest tests.
- Approve or Reject.

## Project Brain Rules

1. Update Project Brain only after user approval.
2. Preserve project history.
3. Preserve lessons learned.
4. Preserve known bugs and fixes.
5. Preserve stable checkpoints.
